🆔 ID GENERATION STRATEGY (from ID_GENERATION_GUIDE.md)
──────────────────────────────────────────────────────────────────────────────────
Hybrid ID System:
  • Employee IDs (NV***): Backend generates NV001, NV002, etc.
  • Customer IDs (KH***): Backend generates KH001, KH002, etc.
  • Rental IDs (THU***): Backend generates THU001, THU002, etc.
  • Borrowing IDs (MUO***): Backend generates MUO001, MUO002, etc.
  • Book Copy IDs (BC***): Backend generates BC001, BC002, etc.
  • Order IDs: Database auto-increment INT (1, 2, 3...)
  • Product IDs: Database auto-increment INT (1, 2, 3...)
  • Membership IDs (MaThe): Manual INT entry (1, 2, 3...)

Benefits: Flexible business logic, user-friendly formatted IDs, fast database lookups
