📊 API ENDPOINT REFERENCE
═══════════════════════════════════════════════════════════════════════════════════

Authentication
  POST   /api/auth/login              Login with credentials
  POST   /api/auth/register           Create new user (admin)
  GET    /api/auth/verify             Verify token validity

Employees (Admin Only)
  GET    /api/employees               List all employees
  POST   /api/employees               Create new employee
  DELETE /api/employees/<id>          Delete employee

Customers (Role-Based Access)
  GET    /api/customers               List all (Admin/Cashier) or own (Customer)
  POST   /api/customers               Create new customer
  GET    /api/customers/me            Get own profile (Customer)
  DELETE /api/customers/<id>          Delete customer (Admin)

Products (All Authenticated Users)
  GET    /api/products                List all products
  POST   /api/products                Create product (Admin)
  DELETE /api/products/<id>           Delete product (Admin)

Transactions
  GET    /api/borrowing               List borrowing (filtered by role)
  GET    /api/rentals                 List rentals (filtered by role)
  GET    /api/stats                   Get statistics (role-specific data)

🔐 SECURITY-FIRST DESIGN
   • JWT tokens instead of session cookies
   • Role-based access at multiple levels (endpoint + data)
   • Password hashing on all stored credentials
   • CORS restriction to prevent unauthorized access

   • Three completely different interfaces for three roles
   • No manual ID entry - all auto-generated

   • Instant role-based redirect after login

📊 SCALABLE ARCHITECTURE
   • Decorator-based access control (easy to add new roles)
   • Data filtering at database level (efficient)
   • Stateless JWT (can scale to multiple API servers)
   • Modular frontend (each dashboard is independent)
